# Sistema de cómputo
Conjunto de componentes (hardware, software, datos, usuarios, procedimientos y comunicaciones) que reciben datos, los procesan y producen información útil. Puede ser un PC, móvil, servidor, servicio en la nube o sistema embebido.
# Hardware
Componentes físicos de un sistema de cómputo, como CPU, memoria, disco y periféricos.
# Software
Programas y códigos que permiten al hardware funcionar; incluye sistemas operativos, aplicaciones, utilitarios y firmware.
# Nube (computación en la nube)
Modelo de entrega de servicios tecnológicos (IaaS, PaaS, SaaS) a través de internet, con características como elasticidad y pago por uso.
# Entrada-Proceso-Salida-Almacenamiento
Modelo básico de los sistemas de cómputo: reciben datos (entrada), los transforman (proceso), generan resultados (salida) y guardan información (almacenamiento).
# API (Interfaz de Programación de Aplicaciones)
Conjunto de reglas que permite la comunicación entre diferentes sistemas o aplicaciones; ejemplo de fuente de entrada de datos.
# IoT (Internet de las Cosas)
Red de dispositivos físicos conectados a internet que pueden enviar y recibir datos; sus actuadores son ejemplos de salida de un sistema.
# Virtualización
Tecnología que permite crear versiones virtuales de recursos de hardware o software, facilitando la flexibilidad y el uso eficiente de los sistemas.
# Ciberseguridad
Conjunto de medidas para proteger sistemas, redes y datos de amenazas; incluye prácticas como cifrado, uso de contraseñas y protección contra phishing.
# Contenedores (Docker)
Herramientas de virtualización ligera que empaquetan aplicaciones y sus dependencias, permitiendo su ejecución en cualquier entorno.
# DevOps / Cloud Engineer
Rol profesional que combina desarrollo de software y operaciones, encargado de gestionar infraestructuras en la nube y procesos de despliegue.
# Ley de Moore
Principio que establece que el número de transistores en un circuito integrado se duplica aproximadamente cada dos años, impulsando la miniaturización y el bajo costo de los sistemas.
# Firmware
Software almacenado en hardware que controla sus funciones básicas; incluye BIOS/UEFI y código embebido en microcontroladores.
# CI/CD (Integración Continua/Despliegue Continuo)
Prácticas de desarrollo moderno que permiten integrar cambios de código y desplegar aplicaciones de forma automatizada y constante
# Sistemas distribuidos
Conjunto de dispositivos independientes que trabajan juntos para ofrecer un servicio común; ejemplo clave es Internet y la Web.
